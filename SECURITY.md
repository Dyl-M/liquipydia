# Security Policy

## Supported Versions

| Version  | Supported |
|----------|-----------|
| Latest   | ✅         |
| < Latest | ❌         |

Only the latest released version of `liquipydia` receives security updates. Users are encouraged to upgrade promptly.

## Reporting a Vulnerability

**Please do not open a public issue for security vulnerabilities.**

Instead, report them privately via one of the following:

- **Email:** [dyl_m.dev@proton.me](mailto:dyl_m.dev@proton.me) — include `[SECURITY]` in the subject line.
- **GitHub Security Advisories:** use
  the [private vulnerability reporting](https://github.com/Dyl-M/liquipydia/security/advisories/new) feature on this
  repository.

### What to include

- A description of the vulnerability and its potential impact.
- Steps to reproduce or a proof of concept.
- The version(s) of `liquipydia` affected.
- Any suggested fix, if applicable.

### What to expect

- **Acknowledgement** within 72 hours.
- **Status update** within 7 days with an assessment and estimated timeline.
- **Credit** in the release notes (unless you prefer to remain anonymous).

## Disclosure Policy

We follow coordinated disclosure. Please allow a reasonable timeframe to address the issue before any public disclosure.
